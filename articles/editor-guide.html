✏️ Как редактировать существующую статью

Способ 1: Через сайт (быстро)

1. Откройте любую статью на Codepedia

2. Прокрутите вниз до подвала страницы

3. Нажмите ссылку "✏️ Редактировать"

4. Вас перенаправит на GitHub

5. Нажмите значок карандаша и внесите правки

6. Напишите краткое описание изменения

7. Нажмите "Commit changes"

Способ 2: Через GitHub

1. Перейдите в репозиторий

2. Откройте папку articles/

3. Найдите нужный файл (например, python.html)

4. Нажмите на него, затем на значок ✏️

5. Внесите изменения и сохраните

✅ Совет: Все изменения сразу применяются к сайту. Если хотите сначала обсудить — создайте Issue.

➕ Как добавить новую статью

Быстрый способ (через GitHub)

1. Перейдите в папку articles/ репозитория

2. Нажмите "Add file" → "Create new file"

3. Назовите: название.html (латиница, например golang.html)

4. Скопируйте шаблон статьи ниже

5. Заполните содержимое

6. Добавьте ссылку на статью в index.html

7. Нажмите "Commit new file"

💬 Система обсуждения статей

Каждая статья может быть обсуждена через GitHub Issues.

Как обсудить статью:

  • Перейдите на вкладку Issues
  • Нажмите "New issue"
  • Выберите тип:
    • 📝 Предложение статьи — для новых тем
    • ✏️ Правка статьи — для исправлений
    • 💬 Обсуждение — для вопросов
  • Напишите название и описание
  • Нажмите "Submit new issue"
💡 Подсказка: Используйте метки (@mentions), чтобы позвать других редакторов.

Где найти обсуждение конкретной статьи?

В подвале каждой статьи есть ссылка "💬 Обсуждение". Она ведёт на Issue, связанный с этой статьёй.

📏 Правила оформления статей

Заголовки

  • h1 — только один на странице, название статьи
  • h2 — основные разделы (добавляются в оглавление)
  • h3 — подразделы внутри h2

Код

  • Короткий код внутри строки: <code>код</code>
  • Блоки кода: <pre><code>...</code></pre>
  • Всегда указывайте язык (через комментарий, если нужно)

Оглавление (TOC)

  • Добавляйте сразу после краткого описания
  • Ссылки должны вести на id разделов
  • Пример: <a href="#features">Особенности</a> и <h2 id="features">✨ Особенности</h2>

Инфоблоки

  • Обычный: <div class="info">...</div>
  • Предупреждение: <div class="warning">...</div>
  • Успех/совет: <div class="success">...</div>

Ссылки

  • На другие статьи Codepedia: <a href="python.html">Python</a>
  • На главную: <a href="../index.html">🏠 Главная</a>
  • Внешние ссылки: <a href="https://..." target="_blank">...</a>

📄 Шаблон статьи

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Название — Codepedia</title>
    <style>
        /* скопируйте стили из существующей статьи */
    </style>
</head>
<body>
    <div class="page">
        <div class="nav">
            <a href="../index.html">🏠 Codepedia</a>
            <a href="python.html">Python</a>
            <a href="cpp.html">C++</a>
        </div>

        <h1>Название статьи</h1>

        <div class="info">
            <strong>📌 Кратко:</strong> Одно-два предложения о чём статья.
        </div>

        <div class="toc">
            <h3>Содержание</h3>
            <ul>
                <li><a href="#section1">Раздел 1</a></li>
                <li><a href="#section2">Раздел 2</a></li>
                <li><a href="#example">Пример кода</a></li>
            </ul>
        </div>

        <h2 id="section1">✨ Раздел 1</h2>
        <p>Текст раздела...</p>

        <h2 id="section2">📖 Раздел 2</h2>
        <ul>
            <li>Пункт 1</li>
            <li>Пункт 2</li>
        </ul>

        <h2 id="example">💻 Пример кода</h2>
        <pre><code>
// ваш код здесь
        </code></pre>

        <div class="footer">
            <a href="#">✏️ Редактировать</a> | 
            <a href="#">📜 История</a> | 
            <a href="#">💬 Обсуждение</a>
        </div>
    </div>
</body>
</html>

💡 Полезные советы

  • Проверяйте код — примеры должны работать
  • Пишите понятно — представьте, что читает новичок
  • Обновляйте дату — в подвале статьи укажите день редактирования
  • Используйте эмодзи — они делают заголовки нагляднее: ✨ 📖 💻 🎯 ⚠️ ✅ ❌
  • Добавляйте ссылки — связывайте статьи друг с другом
⚠️ Важно: Не добавляйте рекламу, спам или оскорбительный контент. Мы — свободная энциклопедия, но с правилами.
